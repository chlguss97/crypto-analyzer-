### 기법 1. Order Block (오더블록) ★ 핵심

**개념:** 기관이 대량 주문을 실행한 가격 영역. 가격이 해당 영역에 돌아오면 반응할 확률이 높다.

```
Bullish OB: 강한 상승 임펄스 직전의 마지막 음봉(하락 캔들)
 ┌──────┐
 │▓▓▓▓▓▓│ ← 이 음봉 = Bullish OB
 └──────┘
    ↑↑↑↑↑↑↑  (강한 상승 임펄스)

Bearish OB: 강한 하락 임펄스 직전의 마지막 양봉(상승 캔들)
    ↓↓↓↓↓↓↓  (강한 하락 임펄스)
 ┌──────┐
 │░░░░░░│ ← 이 양봉 = Bearish OB
 └──────┘
```

**탐지 로직:**
```
1. 임펄스 감지: N봉 내 ATR × 2.0 이상 한 방향 이동
2. 임펄스 시작 직전 반대방향 캔들 식별 = OB 후보
3. OB 영역 = 해당 캔들의 [Low ~ High]
4. 강도 계산:
   - 임펄스 크기 (ATR 배수) → 클수록 강함
   - 거래량 비율 (OB 캔들 vol / 20봉 평균 vol)
   - 상위 TF OB와 겹침 여부 → 겹치면 강도 ×1.5
5. 소진(mitigation) 추적: 가격이 OB를 완전히 관통하면 무효화
```

**BTC 선물 최적 파라미터:**
| 파라미터 | 15m | 1H | 4H |
|---------|-----|----|----|
| 임펄스 기준 | ATR × 1.5 | ATR × 2.0 | ATR × 2.5 |
| 최대 유효기간 | 48시간 | 7일 | 30일 |
| 최대 리테스트 | 3회 | 2회 | 2회 |
| 최소 거래량 비율 | 1.2× | 1.5× | 1.5× |

**시그널 출력:**
```python
{
    'type': 'order_block',
    'direction': 'long',        # 'long' | 'short'
    'strength': 0.82,           # 0~1 (강도)
    'ob_zone': [64200, 64800],  # OB 가격 범위
    'distance_pct': 0.3,        # 현재가 대비 거리(%)
    'htf_aligned': True,        # 상위TF 방향 일치 여부
    'retest_count': 1,          # 리테스트 횟수
}
```

**진입가 산정 — OTE (Optimal Trade Entry):**
```
OB 영역의 50% 지점 (피보나치 0.5 리트레이스먼트)
= (OB_high + OB_low) / 2

예: OB = [64,200 ~ 64,800]
    OTE = 64,500 에 지정가 진입
```

### 기법 2. Fair Value Gap (FVG)

**개념:** 3봉 구조에서 발생하는 가격 갭. 시장이 "공정가"로 되돌아오려는 성질을 이용.

```
Bullish FVG:
  봉1 High ─────┐
                 │ ← 이 갭 = Bullish FVG
  봉3 Low  ─────┘
  (봉1 고가 < 봉3 저가)

Bearish FVG:
  봉3 High ─────┐
                 │ ← 이 갭 = Bearish FVG  
  봉1 Low  ─────┘
  (봉1 저가 > 봉3 고가)
```

**BTC 선물 적용:**
```
- 최소 갭 크기: 0.15% (BTC 변동성 고려, 주식보다 크게)
- FVG + OB 겹침 = "Golden Zone" → 최고 등급 진입 구간
- FVG 50% 채워지면 "부분 소진", 100% 관통하면 무효화
- BTC는 FVG 리테스트 확률 ~65% (4H 기준)
```

**시그널 출력:**
```python
{
    'type': 'fvg',
    'direction': 'long',
    'gap_zone': [63800, 64100],
    'gap_size_pct': 0.47,
    'filled_pct': 0.0,          # 0~1 (채워진 비율)
    'overlaps_ob': True,        # OB와 겹침 여부 ← 핵심
}
```

### 기법 3. Market Structure (BOS / CHoCH)

**개념:** 가격의 스윙 고점/저점 패턴으로 추세 방향과 전환점을 판단.

```
상승 추세 (Bullish):
  HH ──── Higher High
  │
  HL ──── Higher Low (각 저점이 이전보다 높음)

하락 추세 (Bearish):
  LH ──── Lower High
  │
  LL ──── Lower Low (각 고점이 이전보다 낮음)

BOS (Break of Structure): 추세 방향 유지 확인
  상승 중 HH 갱신 = Bullish BOS ✓

CHoCH (Change of Character): 추세 전환 시그널
  상승 중 HL 이탈 = Bearish CHoCH ⚠️ (추세 전환!)
```

**BTC 선물 적용:**
```
- 스윙포인트 감도: strength=5 (좌/우 5봉)
  → BTC 변동성 크므로 너무 작으면 노이즈
- 멀티TF 구조 정렬:
  · 4H 구조 = Bullish + 15m 구조 = Bullish → 강한 롱 시그널
  · 4H 구조 = Bullish + 15m CHoCH Bearish → 역추세 경고
- BOS 후 첫 번째 풀백이 최적 진입 타이밍
- CHoCH는 기존 포지션 청산 시그널로도 사용
```

**시그널 출력:**
```python
{
    'type': 'market_structure',
    'trend': 'bullish',            # 'bullish' | 'bearish' | 'ranging'
    'last_event': 'BOS_bullish',   # 마지막 구조 이벤트
    'last_event_bars_ago': 3,      # 이벤트 이후 봉 수
    'swing_high': 65400,
    'swing_low': 63200,
    'htf_trend': 'bullish',        # 상위TF 추세
    'aligned': True,               # 상하위TF 정렬 여부
}
```

### 기법 4. Bollinger Bands (볼린저 밴드)

**개념:** 이동평균 ± 2σ 밴드. BTC 선물에서 평균회귀 + 스퀴즈 돌파 + 밴드워크 3가지 패턴.

**BTC 선물 3대 패턴:**

```
[패턴 A] 스퀴즈 → 돌파 (Squeeze Breakout)
  밴드 폭(bb_width)이 최근 50봉 최소 → 에너지 축적
  → 돌파 방향으로 진입 (OB/구조와 일치 시)

[패턴 B] 밴드 터치 반전 (Mean Reversion)
  가격이 하단 터치 + RSI < 30 → 롱 반전 기대
  가격이 상단 터치 + RSI > 70 → 숏 반전 기대
  ※ 강한 추세에서는 역효과 → 구조(BOS) 필터 필수

[패턴 C] 밴드워크 (Band Walk)
  가격이 상단 밴드를 따라 이동 = 강한 상승 추세
  → 추세 추종 (풀백 시 중간선 지지에서 추가 매수)
```

**BTC 선물 파라미터:**
```
- period: 20 (표준)
- std_dev: 2.0 (표준)
- bb_position: (현재가 - 하단) / (상단 - 하단)
  · < 0.2: 과매도 영역 (v22 핵심 피처)
  · > 0.8: 과매수 영역
- bb_width: (상단 - 하단) / 중간선
  · < 50봉 최소값의 1.1배: 스퀴즈 상태
- squeeze_duration: 스퀴즈 지속 봉 수 (길수록 돌파 강도 ↑)
```

**시그널 출력:**
```python
{
    'type': 'bollinger',
    'bb_position': 0.12,           # 0~1
    'bb_width': 0.023,             # 밴드폭 비율
    'pattern': 'squeeze',          # 'squeeze' | 'mean_reversion' | 'band_walk'
    'squeeze_bars': 15,            # 스퀴즈 지속 봉수
    'direction': 'long',           # 시그널 방향
    'strength': 0.7,
}
```

### 기법 5. EMA 다중 정배열/역배열

**개념:** 여러 기간 EMA의 정렬 상태로 추세 강도와 방향을 판단.

```
정배열 (Bullish Alignment):
  가격 > EMA9 > EMA21 > EMA50 > EMA200
  → 강한 상승 추세 확인

역배열 (Bearish Alignment):
  가격 < EMA9 < EMA21 < EMA50 < EMA200
  → 강한 하락 추세 확인

크로스 시그널:
  EMA9 × EMA21 골든크로스 → 단기 추세 전환 (빠름)
  EMA50 × EMA200 골든크로스 → 장기 추세 전환 (느림, 신뢰도 높음)
```

**BTC 선물 적용:**
```
- EMA 기간: 9 / 21 / 50 / 200
- 정배열 점수: 
  · 4개 완전 정배열 = 1.0
  · 3개 정렬 = 0.75
  · 2개 이하 = 0.0 (추세 불명)
- EMA 거리(dist): 현재가와 EMA 사이 거리(%)
  · EMA200 dist > 10%: 과도한 확장 → 평균회귀 경고
  · EMA200 dist < 1%: 핵심 지지/저항 테스트 중
- BTC 24/7 특성: EMA200(4H)이 일봉 EMA50과 유사한 역할
```

**시그널 출력:**
```python
{
    'type': 'ema',
    'alignment': 'bullish',      # 'bullish' | 'bearish' | 'mixed'
    'alignment_score': 1.0,      # 0~1
    'ema9': 64520,
    'ema21': 64100,
    'ema50': 63200,
    'ema200': 61800,
    'dist_ema200_pct': 4.2,      # 현재가 대비 EMA200 거리(%)
    'recent_cross': 'golden_9_21',  # 최근 크로스 이벤트
    'cross_bars_ago': 5,
}
```

### 기법 6. RSI + 다이버전스

**개념:** RSI 과매수/과매도 + 가격-RSI 다이버전스로 추세 전환/지속 판단.

```
[일반 다이버전스] — 추세 전환 시그널
  Bullish: 가격 LL + RSI HL → 하락 모멘텀 약화 → 반등 예상
  Bearish: 가격 HH + RSI LH → 상승 모멘텀 약화 → 하락 예상

[히든 다이버전스] — 추세 지속 시그널  
  Bullish: 가격 HL + RSI LL → 상승 추세 중 건전한 조정
  Bearish: 가격 LH + RSI HH → 하락 추세 중 약한 반등
```

**BTC 선물 적용:**
```
- RSI 기간: 14 (표준) + 7 (단기 보조)
- 과매수/과매도 기준:
  · RSI > 70: 과매수 (숏 경고)
  · RSI < 30: 과매도 (롱 기회)
  · RSI > 80: 극단 과매수 (선물에서 특히 위험 — 청산 캐스케이드)
  · RSI < 20: 극단 과매도 (바닥 가능성)
- BTC 특수 고려:
  · 강한 추세에서 RSI 50이 지지/저항 역할
  · RSI 40~60 = 중립 (방향 판단 보류)
  · 다이버전스는 최소 5봉 이상 간격 (노이즈 필터)
- v22 교훈: RSI + BB position 조합이 바닥 감지에 효과적
  (bb_position < 0.2 + RSI < 35 = 역발상 매수 조건)
```

**시그널 출력:**
```python
{
    'type': 'rsi',
    'rsi_14': 28.5,
    'rsi_7': 22.1,
    'zone': 'oversold',          # 'overbought' | 'neutral' | 'oversold'
    'divergence': 'bullish',     # 'bullish' | 'bearish' | 'hidden_bullish' | 'hidden_bearish' | None
    'divergence_strength': 0.8,  # 다이버전스 강도 0~1
    'direction': 'long',
}
```

### 기법 7. 거래량 패턴 분석

**개념:** 거래량의 급증/감소 패턴으로 돌파 진위, 추세 강도, 고점/저점 판단.

```
[패턴 A] 볼륨 스파이크 (Volume Spike)
  현재 거래량 / 20봉 평균 거래량 = spike_ratio
  · spike > 3.0: 비정상적 관심 (돌파 or 캐스케이드)
  · spike > 5.0: 극단적 이벤트 (v22 전문모델 기준)

[패턴 B] 볼륨 드라이업 (Volume Dry-up)
  거래량 3봉 연속 감소 + 가격 횡보 = 에너지 축적
  → BB 스퀴즈와 함께 출현하면 돌파 임박

[패턴 C] 볼륨 다이버전스
  가격 신고가 + 거래량 감소 = 약한 돌파 (페이크 가능)
  가격 신저가 + 거래량 감소 = 매도 소진 (바닥 가능)

[패턴 D] 클라이맥스 볼륨 (Climax Volume)
  극단적 거래량 + 긴 꼬리 캔들 = 추세 종료 시그널
  · 윗꼬리 + 대량거래 = 매수 클라이맥스 (고점)
  · 아랫꼬리 + 대량거래 = 매도 클라이맥스 (저점)
```

**BTC 선물 전용 — Taker Buy/Sell Ratio:**
```
- taker_buy_volume / total_volume = taker_ratio
  · > 0.55: 공격적 매수 (시장가 매수 우위)
  · < 0.45: 공격적 매도 (시장가 매도 우위)
- OKX API에서 직접 제공
```

**시그널 출력:**
```python
{
    'type': 'volume',
    'spike_ratio': 4.2,
    'pattern': 'spike',           # 'spike' | 'dryup' | 'divergence' | 'climax'
    'taker_buy_ratio': 0.58,      # 테이커 매수 비율
    'trend_confirm': True,        # 추세 방향과 거래량 일치 여부
    'direction': 'long',
    'strength': 0.75,
}
```

### 기법 8. Funding Rate (펀딩비) — 선물 전용 ★

**개념:** 무기한 선물의 가격을 현물에 수렴시키기 위한 주기적 수수료. 시장 과열/과냉 지표.

```
펀딩비 > 0 (양수): 롱이 숏에게 지불 → 롱 과열
펀딩비 < 0 (음수): 숏이 롱에게 지불 → 숏 과열

[핵심 전략: 극단 펀딩비 역발상]
  펀딩비 > +0.05%: 롱 과밀 → 숏 스퀴즈 위험 낮음, 롱 청산 위험 ↑
  펀딩비 > +0.1%:  극단 과열 → 역방향(숏) 진입 기회
  펀딩비 < -0.05%: 숏 과밀 → 숏 스퀴즈 가능 → 롱 기회
  펀딩비 < -0.1%:  극단 공포 → 역방향(롱) 진입 기회
```

**BTC 선물 적용:**
```

- 정상 범위: -0.01% ~ +0.01%

- 극단 범위: |funding| > 0.05% → 역발상 시그널 활성
```

### 기법 9. Open Interest (미결제약정) — 선물 전용 ★

**개념:** 열려있는 선물 포지션의 총 규모. 시장 참여도와 레버리지 수준을 나타냄.

```
[OI + 가격 조합 해석]

  가격 ↑ + OI ↑ = 새로운 롱 유입 → 추세 강화 확인 ✓
  가격 ↑ + OI ↓ = 숏 청산(커버링) → 약한 상승 (곧 소진) ⚠️
  가격 ↓ + OI ↑ = 새로운 숏 유입 → 하락 추세 강화 확인 ✓
  가격 ↓ + OI ↓ = 롱 청산(손절) → 약한 하락 (바닥 근접) ⚠️

[OI 다이버전스]
  가격 신고가 + OI 감소 = 신규 매수 없음 → 가짜 돌파 경고
  가격 신저가 + OI 감소 = 투매 마무리 → 반등 가능
  
[OI 급증 (Spike)]
  OI 24h 변화 > +10%: 대규모 레버리지 유입 → 변동성 폭발 예고
  · 방향 판단은 펀딩비 + 롱숏비율과 교차 확인
```

**BTC 선물 수치 기준 (2026년):**

```
  · ±10%/1H: 경고 (캐스케이드 가능)
```

### 기법 10. Liquidation Level (청산 구간) — 선물 전용 ★

**개념:** 레버리지 포지션이 강제 청산되는 가격대. 청산 캐스케이드가 가격을 급격히 밀어냄.

```
[청산 캐스케이드 메커니즘]
  가격 하락 → 롱 청산(강제 매도) → 추가 하락 → 더 많은 롱 청산 → 폭락
  가격 상승 → 숏 청산(강제 매수) → 추가 상승 → 더 많은 숏 청산 → 폭등

[Liquidation Heatmap 활용]
  청산 밀집 구간 = "자석 가격" → 가격이 끌려감
  · 위쪽 숏 청산 밀집 → 가격이 올라가 "꿀 따먹기" (숏 스퀴즈)
  · 아래쪽 롱 청산 밀집 → 가격이 내려가 "꿀 따먹기" (롱 스퀴즈)

[전략: 청산 구간 반대편 진입]
  롱 청산 밀집대 터치 후 반등 시 → 롱 진입 (매도 압력 소진)
  숏 청산 밀집대 터치 후 하락 시 → 숏 진입 (매수 압력 소진)
```

**BTC 선물 적용:**
```
- 추정 청산가 계산: entry_price × (1 ± 1/leverage)
  · 10x 롱: 진입가 -10% 에서 청산
  · 25x 롱: 진입가 -4% 에서 청산
- 주요 레버리지 구간별 청산대 매핑:
  · 5x, 10x, 25x, 50x, 100x
- OI + 레버리지 분포로 청산 밀집 구간 추정
- 외부 API: CoinGlass Liquidation Heatmap (보조)
```

```python
    'long_liq_density': 0.7,          # 롱 청산 밀집도 (0~1)
```

### 기법 11. Long/Short Ratio (롱숏 비율) — 선물 전용

```
[역발상 원칙]

  롱/숏 비율 < 0.5: 숏 과밀 → 상승 반전 기회 (군중이 틀림)
  롱/숏 비율 0.8~1.2: 균형 → 방향 판단 보류

[두 가지 지표]

  → 두 지표 불일치 시: 고래가 개인과 반대 포지션 = 고래 추종
```

### 기법 12. CVD (Cumulative Volume Delta)

**개념:** 시장가 매수량 - 시장가 매도량의 누적. 공격적 참여자의 실제 방향.

```
[CVD 다이버전스] — 가장 강력한 시그널

  가격 ↑ + CVD ↓ = 매수 주도 아님 → 약한 상승 (곧 꺾임)
  가격 ↓ + CVD ↑ = 매도 주도 아님 → 약한 하락 (바닥 근접)

[CVD + OB 결합]
  OB 리테스트 + CVD 반전 상승 = 기관 매수 확인 → 고확률 롱
  OB 리테스트 + CVD 계속 하락 = 지지 실패 → OB 무효화 가능
```

**BTC 선물 적용:**
```
- CVD 데이터: OKX 체결(trades) WebSocket에서 실시간 계산
- aggressor 판단: 체결가 >= ask면 taker buy, <= bid면 taker sell
- CVD 기간: 15m/1H/4H 별도 누적 트래킹
- CVD + 가격 다이버전스가 OB/FVG 영역에서 발생하면 신뢰도 ×2
```

**시그널 출력:**

```python
    'price_cvd_divergence': True,  # 가격-CVD 다이버전스
    'divergence_type': 'bullish',  # 'bullish' | 'bearish'
    'direction': 'long',
```

### 기법 13. ATR 기반 동적 SL/TP

**개념:** 변동성에 비례하는 손절/목표가 설정. BTC 변동성 변화에 자동 적응.

```
[SL 산정]
  롱 SL = 진입가 - ATR(14) × 1.5
  숏 SL = 진입가 + ATR(14) × 1.5
  · 최소 SL: 0.5%  (너무 촘촘하면 노이즈에 청산)
  · 최대 SL: 3.0%  (너무 넓으면 리스크 과다)

[TP 산정 — RR 비율 기반]
  TP1 = 1.5 × SL 거리 (50% 물량 청산)
  TP2 = 2.5 × SL 거리 (25% 물량 청산)  
  TP3 = 트레일링 (나머지 25%)

[포지션 사이즈]
  사이즈 = (계좌 × 리스크비율) / (SL거리% × 레버리지)
  · 리스크비율 = 2% (1회 매매 최대 손실)
  
  예: 계좌 $10,000 / 리스크 2% / SL 1.5% / 레버 5x
      사이즈 = (10000 × 0.02) / (0.015 × 5) = $2,667
```

**BTC 선물 ATR 참고치:**
```
  15m ATR: ~$150~400 (시장 상황에 따라)
  1H ATR:  ~$300~800
  4H ATR:  ~$600~1,500
  1D ATR:  ~$1,500~4,000
```

## Part 4. 트레일링 & 청산 전략

### 트레일링 스톱 (v22 CheckTrailing 이식 + 확장)

```
Tier 0: 진입 직후

Tier 1: 수익 +1.0% 도달
  SL → 진입가 (본전)

Tier 4: 수익 +5.0% 이상
```
